## Demo Dry Runs (20 min per team)

Each team gets 20 minutes:
- 12 min demo
- 8 min facilitator feedback

**Feedback areas:**
- Clarity of problem statement
- Live demo execution (did it work?)
- Claude features demonstrated
- Code quality and architecture
- Team collaboration visible
- Storytelling and pacing

### Peer Code Review (60 min)

**Format:** Each team reviews one other team's repo.

Review checklist:
- CLAUDE.md quality: Is it useful? Would a new developer benefit?
- Code organization: Clean structure, separation of concerns
- Claude integration: API usage patterns, error handling, cost efficiency
- Testing: Are there tests? Do they cover key paths?
- Security: API key handling, input validation, prompt injection defense
- Documentation: README, inline comments where needed

---

## Facilitator Notes

- Dry runs should simulate the real demo environment (screen share, time limits).
- Give specific, actionable feedback — "your intro is 4 minutes, cut to 2" not "be more concise."
- Case studies will be shared internally — quality matters. Review drafts before Day 10.
