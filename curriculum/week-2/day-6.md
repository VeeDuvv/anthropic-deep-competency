## Session Details

### AI Product Process Deep Dive (45 min)

Expanding on Day 5's intro with real examples:

- **Strategy:** Identifying AI-appropriate problems. The "would a human with enough time solve this?" test.
- **Spec:** Claude-specific spec writing: what to specify, what to leave flexible, how to define "done."
- **Harness:** CLAUDE.md as the single source of truth. Test-driven development with Claude Code.
- **Build:** Sprint cadence with Claude Code. When to plan mode vs. just build. Code review patterns.
- **Demo:** Storytelling for technical demos. Problem → Solution → Impact.
- **Comms:** Writing internal case studies that drive adoption.

### Writing Effective AI Specs (45 min)

**Workshop format:** Facilitator writes a spec live, participants critique and improve.

Spec template:
- Problem statement (1 paragraph)
- Target user + persona
- Functional requirements (numbered, testable)
- Non-functional requirements (performance, security, scale)
- Claude features to use (from Week 1)
- Acceptance criteria
- Out of scope (explicitly)
