## Session Details

### Competency Knowledge Check (60 min)

**Format:** Individual, open-book, timed

- 40 questions across 5 domains (8 per domain)
- Mix of multiple choice, short answer, and code snippets
- Not graded pass/fail — used to identify gaps and guide Week 2 focus
- Results shared privately with each participant

**Domains covered:**
1. Claude Fundamentals (architecture, models, capabilities)
2. Prompt Engineering (techniques, patterns, evaluation)
3. Tool Use and Agents (tool definition, agentic loops, Agent SDK)
4. Enterprise Integration (RAG, MCP, embeddings, classification)
5. Safety and Responsible AI (guardrails, injection defense, evaluation)

### AI Product Process (45 min)

**The framework:**

1. **Strategy** — What problem are we solving? For whom? Why now? Why AI?
2. **Spec** — Requirements, constraints, acceptance criteria, tech stack
3. **Harness** — CLAUDE.md, repo setup, CI/CD, testing strategy
4. **Build** — Sprint-based development with Claude Code as the primary tool
5. **Demo** — Present to stakeholders: what it does, how it works, what we learned
6. **Comms** — Internal write-up: lessons, patterns, reusable components

### Team Formation + Sprint 0 (105 min)

- 3 teams of 4 participants (mixed personas per team)
- Each team gets a project brief with:
  - Problem statement
  - Target user
  - Required Claude features (must use at least 3 from Week 1)
  - Demo criteria
- Sprint 0 deliverables:
  - GitHub repo with CLAUDE.md
  - Architecture decision record (ADR)
  - Sprint 1 backlog (5-8 items)
